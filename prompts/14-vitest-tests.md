# Prompt 14 — Vitest Test Suite

## User
could you please test the functionality of my angular app with vitest

## Assistant
Added a comprehensive Vitest test suite covering all three core services, and fixed two pre-existing failures in `app.spec.ts`.

### New spec files

**`src/app/services/openapi-parser.service.spec.ts`** (47 tests)
- `detectFormat` — JSON object/array detection vs YAML fallback
- `toPascalCase` / `toCamelCase` — kebab, snake_case, single word, already-pascal input
- `parse` metadata — title/version extraction from JSON and YAML; defaults when `info` is absent
- Object schemas — properties, required flags, PascalCase name conversion, nullable flag
- Enum schemas — `kind: 'enum'`, `enumValues` population
- `allOf` merging — properties and required arrays merged from `$ref` and inline schemas
- Type mapping — all primitives (`string`, `int`, `long`, `float`, `double`, `bool`), `$ref` → `ref`, arrays, inline enums
- Path parsing — operation grouping by tag, all HTTP methods, `operationId` passthrough and auto-generation, path/query parameters, request body and response type extraction, `Default` tag fallback

**`src/app/services/csharp-generator.service.spec.ts`** (27 tests)
- `toCsType` — all primitive mappings, required vs optional (`?`), arrays (`IReadOnlyList<T>`), ref types, enum → `string`, any → `object`
- `generateDtos` — `// <auto-generated />` header, namespace from title, record with required/nullable params, enum with values, empty record
- `generateDtoFiles` — one file per schema, correct `filename` and `path`, namespace and content per file
- `generateControllers` — `using` directives, `[ApiController]`, HTTP verb attributes, sub-routes for path params, `[FromRoute]`/`[FromBody]` params, `IActionResult` vs `ActionResult<T>`, XML doc `<summary>`, `NotImplementedException` body
- `generateControllerFiles` — one file per tag, correct `filename` and `path`
- Namespace derivation — spaces → PascalCase, empty title → `MyApi`

**`src/app/services/typescript-generator.service.spec.ts`** (29 tests)
- `toZodType` — all primitive Zod mappings, ref with/without inline enum values, inline enum, arrays (`z.array(...)`), `.nullable()` modifier
- `toTsType` — all primitive TypeScript mappings, ref, inline enum union, arrays
- `generateSchemas` — header + `import { z }`, enum schema, object schema with all property types, inlined enum refs, array Zod type, empty object schema
- `generateSchemaFiles` — one file per schema + barrel `index.ts`, correct filenames and paths, cross-schema imports for object refs, no import for inlined enum refs
- `generateServices` — Angular/RxJS imports, `@Injectable` class, GET (list + by-id), POST, PUT, DELETE methods with correct signatures, URL template literals for path params, JSDoc summary
- `generateServiceFiles` — one file per tag, correct filename and path, `import type * as Schemas from '../schemas'`

### Fixed `src/app/app.spec.ts`
- Added `window.matchMedia` stub (jsdom does not implement it; the `App` constructor calls it to detect the system colour scheme preference)
- Updated the stale title assertion from a non-existent `<h1>` to the actual `.toolbar-title` span
